Can't believe how much I did these days. Today is 29/09/2025 DD/MM/YYYY, and I made IDTs to my new kernel, published it on GitHub open-source, adopted a new package and made a package on the AUR. About the new package adoption, I am now officially the new maintainer of a package called autotrace-git. It's a package made to convert bitmap images into vector ones. It's the only autotrace package orphaned, and I chose this package purely randomly. Now I might not have time to spend on it, but I will try to revisit it as often as I can!

I'm really happy that the recentmost AUR project, ttybatd, has not been flagged with a deletion request, compared to my first package which was flagged immediately and removed within a week because it was too useless.

Though it's still really niche, I'm happy with it.

I am currently installing Gentoo Linux, and it is the world's hardest Linux distribution to use. A lot of parts for the (manual) installation is very similar to Arch's hands-on installation, but some parts like the stage3 tarball are completely different. Gotta say, I spent 12 hours for the installation and it is not done. You see, on Gentoo, everything is compiled from source code (code developers can read) to bytecode/binary code (code machines can read) and it is the whole philosophy of Gentoo. However, compiling them (especially in a virtual machine like me) takes an enormous amount of time and needs patience.

I finally successfully made my first kernel! This kernel is written in C and prints a simple message of "GTechOS is live!". It's a very starter kernel project but I see it a big achievement.

GTechOS will be listed as an official project on this site soon.

I'm starting on OSDev and trying to make a functional kernel. I know, OSDev's one of the hardest branches in IT, but Linus is the same person who made a kernel when he was in University of Helsinki. My plan is to ask AI to generate code, then I learn about C/NASM while building up that OS. It's one job that's really hard, and it's part of my mission of providing a unified system of libre hardware and software.
